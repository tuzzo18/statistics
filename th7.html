Th7: "Online" Algorithms (Data Streams): Ideas and code

Online algorithms are algorithms that process data as it becomes available, making decisions or providing solutions without having the entire input in advance. These algorithms are particularly relevant in scenarios where data is continuously streaming or arriving incrementally.
It means that an online algorithm processes its input piece-by-piece in a serial fashion, i.e., in the order that the input is fed to the algorithm, without having the entire input available from the start.

Because it does not know the whole input, an online algorithm is forced to make decisions that may later turn out not to be optimal, and the study of online algorithms has focused on the quality of decision-making that is possible in this setting.

Online Problems and Algorithms

  • Online Paging: In the online paging problem, a sequence of page requests is processed, and the algorithm must decide which pages to keep in memory (of fixed-size), with the goal of minimizing page faults (when a requested page is not in memory).

    •                         
Algorithm: Online Paging

Input: Sequence of page requests
        Cache size (number of pages that can be held in memory)

Initialize an empty cache

For each page request in the sequence:
    If the page is already in the cache:
        Do nothing (cache hit)
    Else:
        If the cache is full:
            Remove the least recently used page from the cache
        Add the requested page to the cache

End Algorithm
  • Online Bin Packing: In the online bin packing problem, items of varying sizes arrive one at a time, and the goal is to pack them into bins efficiently, i.e., minimizing the number of bins used.

    •     
Algorithm: Online Bin Packing

Input: Sequence of items with sizes
        Bin capacity (maximum size a bin can hold)

Initialize an empty set of bins

For each item in the sequence:
    Try to pack the item into an existing bin (first fit, best fit, etc.)
    If the item doesn't fit in any existing bins:
        Open a new bin and place the item in it

End Algorithm
  • Online Scheduling: In online scheduling, tasks arrive over time, and the algorithm must decide how to schedule them to optimize some criteria (e.g., minimizing completion time).
These pseudocodes provide a high-level overview of the algorithms. The specific implementation details, such as the method for selecting which page to evict in the paging algorithm or the bin-packing strategy, may vary based on the specific requirements and variations of these algorithms. Additionally, more sophisticated algorithms may exist for these problems, and the choice of algorithm depends on the specific characteristics of the problem at hand.

Online algorithms are contrasted with offline algorithms, which have access to the entire input dataset before processing begins.

Competitive analysis is a common framework for evaluating the performance of online algorithms. It compares the performance of an online algorithm to an optimal offline algorithm, which has knowledge of the entire input sequence. The competitive ratio is a measure of how well the online algorithm performs compared to the offline optimal algorithm.

Research in online algorithms continues to evolve, addressing new challenges arising in dynamic and streaming data environments. Advances in this field contribute to more efficient and responsive algorithms for real-world applications where data arrives continuously.
